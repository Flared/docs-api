Flare's API is available at `https://api.flare.io`.
Requests are authenticated with **API tokens** which can be obtained using an **API key**.

## Authentication Objects and Purpose

<CardGroup cols={1}>

<Card title="API Key" icon="key">
    Used to generate API tokens. Does not expire.
</Card>

<Card title="API Token" icon="file-shield">
    Used to access API endpoints. Expire after 1 hour.
</Card>

<Card title="Refresh Token" icon="arrows-rotate">
    Used to obtain new API tokens without the API key.
</Card>

</CardGroup>

### 1 - Obtaining an API key

Users can generate API keys by visiting the [Profile page](https://app.flare.io/#/profile) under the "API Keys" section.

<Frame
  caption="Sidebar -> Configure -> Profile -> API Keys (bottom)"
  className="flex justify-center"
>
  <img width="70%" src="/images/introduction/authentication/api-key-create.png" />
</Frame>

API keys are associated to a user and will have the same permissions as the user that generated them:
- If you can access a tenant, the API key will have access to that tenant.
- If you are an organization administrator, the API key will have organization administrator access.


### 2 - Obtaining an API Token

API tokens are obtained using Flare API keys. They can be generated using the [tokens/generate](/api-reference/tokens/endpoints/generate) endpoint.

### 3 - Access an API endpoint

API tokens may then be used using by specifying the `Authentication Bearer <api-token>` header.

API tokens can be tested using the [tokens/test](/api-reference/tokens/endpoints/test) endpoint.

<CodeGroup>

```bash cURL example
curl --request GET \
  --url https://api.flare.io/tokens/test \
  --header 'Bearer: <api-token>'
```

</CodeGroup>


## Refreshing an API Token

API tokens expire after one hour. After this delay, you must do one of the following:
- 1. Generate a new token using the `tokens/generate` endpoint: **This works for most use cases** if your program still has access to the Flare API key.
- 2. Generate a new tokens using the `tokens/refresh` endpoint: This is useful for programs that don't keep the API key.

See the documentation for the [tokens/refresh](/api-reference/tokens/endpoints/refresh) endpoint.
