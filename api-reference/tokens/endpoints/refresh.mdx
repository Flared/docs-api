---
title: 'Refresh an API Token'
api: 'POST https://api.flare.io/tokens/refresh'
authMethod: 'none'
---

This endpoint can be used to obtain a new API token using the refresh token provided in cookies.

It returns a payload that is identical to the [tokens/generate](/api-reference/tokens/endpoints/generate) endpoint.

## Headers

<ParamField header="Cookie" type="string" placeholder="refresh_token=..." required>
  The cookie that was set by the `tokens/generate` endpoint.
</ParamField>

## Example Usage

<CodeGroup>

```bash cURL
# Fetch initial token and save refresh
# token to my-cookies.jar
curl \
    --request POST \
    --user ":<api-key>" \
    --cookie-jar "my-cookies.jar" \
    https://api.flare.io/tokens/generate

# Refresh using the cookie jar.
curl \
    --request POST \
    --cookie "my-cookies.jar" \
    https://api.flare.io/tokens/refresh
```

```python Python
import requests
from requests.auth import HTTPBasicAuth

session = requests.Session() # This will remember cookies.
api_key = input("Enter your Flare API key:\n")

resp = session.post(
  "https://api.flare.io/tokens/generate",
  auth=HTTPBasicAuth('', api_key)
)
api_token = resp.json()["token"]
print(f"Token: {api_token}")

# We don't need to pass any parameters here because the
# requests session will automatically forward cookies.
resp = session.post(
  "https://api.flare.io/tokens/refresh",
)
new_api_token = resp.json()["token"]
print(f"New token: {new_api_token}")
```
